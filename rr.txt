B2. Pipeline Changes for Ticket/How-to Auto-creation
Fix in discovery step (P1.8) or page-plan step (P1.9): When discovery creates proposed tickets, also create a skeleton how-to doc in kb2_howto linked to the ticket. When page-plan creates entity page plans for new projects, ensure discovered projects get both an entity page plan AND a linked how-to doc.
About the data model: The linking is one-directional — howtos have ticket_id and project_node_id fields pointing to tickets and projects. Tickets and projects themselves don't have fields pointing back to howtos or each other. If you want bidirectional linking stored in the DB (e.g. howto_id on tickets, project_node_id on tickets), those fields would need to be added to the pipeline that creates/updates tickets. Right now, the UI derives the reverse lookups from the howto data.
