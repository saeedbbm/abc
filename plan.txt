Parsed Documents (MongoDB) ← THE single source of truth
    │
    ├──→ Vector Index (Qdrant)     — each chunk links back to doc_id + offset
    │
    └──→ Knowledge Graph (MongoDB) — each node links back to doc_id via source_refs
              │
              └──→ AI Pages        — each item links to node_id + source chunk_ids
                      │
                      └──→ Human Pages — each paragraph links to AI page item_ids


When a human verifies or edits something, the change should flow like this:
Human edits via Verify Card    → updates the Claim    → updates the AI page item (traced via claim_id)    → regenerates/updates the Human page paragraph (traced via AI page item link)    → re-embeds the changed content in Qdrant (traced via page_id)
And when new raw input arrives (e.g., new Confluence page):
New parsed document added    → new vector chunks created in Qdrant    → entity extraction finds new/updated entities    → graph updated    → affected AI pages regenerated    → affected human pages regenerated    → new embeddings for updated pages
