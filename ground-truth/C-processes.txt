PROCESS 1: PR Review and Merge Workflow
================================================================================

Name:       PR Review and Merge Workflow
Status:     active
Description:
  1. Developer creates a feature branch from main
  2. Opens a PR with a description of what was built and why
  3. Gets at least one approval (Matt or Tim for most things)
  4. Squash merge to main
  5. CI/CD auto-deploys to staging
  6. Manual promotion from staging to production

Sources:
  Confluence:   YES — "PawFinder Engineering Onboarding Guide" PR Process section
  Jira:         NO
  GitHub:       YES — every PR in the data follows this workflow
  Slack:        NO (not discussed as a process, just practiced)

PROCESS 2: CI/CD Pipeline
================================================================================

Name:       CI/CD Pipeline (GitHub Actions)
Status:     active

PROCESS 3: Shelter Data Sync
================================================================================

Name:       Shelter Partner Data Sync
Status:     active
Description:
  1. Scheduler triggers sync every 15 minutes
  2. For API shelters: fetch from their endpoints
  3. For CSV shelters: process new uploads since last sync
  4. Normalize data to internal schema
  5. Diff against current database
  6. Apply inserts, updates, soft-deletes
  7. Invalidate relevant caches
  Process is idempotent. Matt gets alerted if 3+ consecutive failures
  for any shelter.

Sources:
  Confluence:   YES — "Shelter Partner Integration Spec" (full description)
  Jira:         NO
  GitHub:       NO
  Slack:        NO

PROCESS 4: Database Migration Process
================================================================================

Name:       Database Migration Process
Status:     active
Description:
  - Migrations live in /db/migrations in the API repo, timestamped
  - Run with: npm run db:migrate
  - Create new migration: npm run db:migrate:create -- --name name
  - Post-incident rule: any migration on tables over 10K rows must be
    reviewed by Matt before deploying
  - CI pipeline flags ALTER TABLE statements on large tables

Sources:
  Confluence:   YES — "PawFinder Engineering Onboarding Guide" Database section,
                "Incident Postmortem" Resolution section
  Jira:         YES — PAW-29 (postmortem task)
  GitHub:       NO
  Slack:        YES — #engineering 2024-01-22 (Jack asks where migrations are,
                Matt points to /db/migrations)
